## 1. 项目概述

### 1.1 项目名称（建议）

首发建议使用更中性的名称，避免“预测 / 荐号 / 高分号码”语义：

- 双色球历史开奖档案
- 双色球历史数据查询
- 双色球开奖形态统计
- 双色球开奖档案馆

### 1.2 项目定位

本产品是一个基于双色球历史开奖数据的**事实型查询与形态统计工具**，用于为用户提供：

- 历史开奖数据查询
- 用户输入号码的历史形态识别
- 常规形态与特殊形态的历史档案展示
- 历史遗漏、分布、频率、时间轴等统计可视化

**产品不提供预测、荐号、选号建议、中奖承诺或购彩引导。**

### 1.3 项目目标

在微信小程序场景下，做一个：

- 用户能快速看懂
- 能持续浏览和探索
- 以历史事实展示为主
- 审核语义尽量稳健
- 便于后续扩展为“开奖档案馆 / 形态博物馆”的轻量产品

### 1.4 目标用户

- 对双色球历史开奖数据有浏览兴趣的普通用户
- 喜欢查看号码形态、冷知识、稀有形态的娱乐型用户
- 希望查询某组号码触发过哪些历史形态的用户

### 1.5 核心价值

- 把“开奖数据”变成“可浏览的统计档案”
- 把“用户输入号码”变成“可解释的形态识别结果”
- 把“单次查询工具”升级为“可停留、可探索、可回看”的历史数据产品

## 2. 产品边界与合规原则

### 2.1 产品边界

本产品只做以下内容：

1. 历史开奖数据展示
2. 历史统计结果展示
3. 用户输入号码的历史形态识别
4. 历史频率 / 历史遗漏 / 历史分布 / 最近出现时间展示
5. 稀有形态与常规形态的档案化展示

本产品明确不做以下内容：

1. 号码预测
2. 荐号
3. 选号建议
4. 中奖率承诺
5. 付费预测服务
6. 导流购彩
7. 互联网售彩
8. “高分号码生成”“推荐号码生成”“命中率优化”等功能

### 2.2 合规展示原则

所有页面均需遵守以下原则：

- 仅展示历史事实，不推导未来开奖
- 仅展示历史相对位置，不表达“即将出现 / 该回补了 / 值得关注”
- 不使用“提高中奖率 / 更可能中奖 / 更值得买 / 更稳 / 更准”等暗示性表述
- 不将遗漏、高频、低频包装为选号建议

### 2.3 固定免责声明

所有核心页面（首页、号码分析页、形态详情页、形态博物馆页）固定展示：

> 本产品仅基于历史开奖数据做事实统计与可视化展示，仅供娱乐参考，不构成任何购彩建议。彩票开奖具有随机性和独立性，历史数据不代表未来结果。

### 2.4 未成年人保护

- 不面向未成年人提供与购彩相关的引导性内容
- 不出现鼓励频繁购彩、冲动购彩的表达

## 4. 数据准备

### 4.1 数据来源

- 中国福利彩票官方公开开奖数据
- 官方授权公开接口或可核验数据源

### 4.2 基础数据结构

每条开奖数据应包含：

| 字段名 | 类型 | 说明 |
|---|---|---|
| `issue` | String | 期号，如 `2026030` |
| `draw_date` | Date | 开奖日期 |
| `red_balls` | List[int] | 6个红球，升序存储 |
| `blue_ball` | Int | 1个蓝球 |

### 4.3 数据校验

即使来源为官网，也保留最小校验逻辑：

1. 期号去重
2. 红球数量必须为6
3. 红球范围必须在1-33且不得重复
4. 蓝球范围必须在1-16
5. 红球统一升序存储
6. 数据入库前保留校验日志

### 4.4 数据存储建议

建议拆为两层：

#### 4.4.1 原始开奖表 `draw_results`

存储原始开奖结果。

#### 4.4.2 统计聚合表 `shape_stats`

预计算各类形态的历史统计结果，避免查询时全量扫描。

#### 4.4.3 形态命中表 `draw_shape_map`

记录每期开奖命中了哪些形态，便于反查最近出现时间与时间轴。

## 5. 形态体系定义

### 5.1 红球基础分布类

1. 大小比：1-16为小，17-33为大，统计 0:6 ~ 6:0
2. 奇偶比：统计 0:6 ~ 6:0
3. 标准三区间分布：1-11 / 12-22 / 23-33
4. 断区：断1区 / 断2区
5. 012路分布
6. 尾数分布：同尾号、全不同尾、同尾组数

### 5.2 红球组合结构类

1. 连号细分：无连号、1组2连号、1组3连号、2组2连号、3连+2连、全6连号等
2. 重号 / 邻号（相对上一期）
3. 和值区间
4. 跨度区间
5. AC值

### 5.3 蓝球专属形态

1. 蓝球大小
2. 蓝球奇偶
3. 蓝球012路
4. 蓝球当前遗漏

### 5.4 红蓝联动形态

1. 红蓝同号
2. 红蓝属性匹配（奇偶 / 大小 / 012路）
3. 红蓝差值相关形态

### 5.5 特殊低频形态

作为“形态博物馆”重点展示对象，包括但不限于：

1. 全6连号
2. 全同尾
3. 全奇 / 全偶
4. 极端断区
5. 极低和值 / 极高和值
6. 稀有 AC 值
7. 罕见复合连号结构

## 6. 统计口径定义

### 6.1 基础统计字段

对每个形态统一维护以下字段：

| 字段名 | 类型 | 说明 |
|---|---|---|
| `shape_type` | String | 形态类别，如 `size_ratio` |
| `shape_key` | String | 形态标识，如 `3:3` / `six_consecutive` |
| `shape_name` | String | 展示名称 |
| `occurrence_count` | Int | 历史出现次数 |
| `occurrence_rate` | Float | 历史出现频率 |
| `first_seen_issue` | String | 首次出现期号 |
| `first_seen_date` | Date | 首次出现日期 |
| `last_seen_issue` | String | 最近一次出现期号 |
| `last_seen_date` | Date | 最近一次出现日期 |
| `current_gap` | Int | 当前距上次出现的期数 |
| `avg_gap` | Float | 历史平均间隔 |
| `median_gap` | Float | 历史中位间隔 |
| `p80_gap` | Float | 历史80分位间隔 |
| `p90_gap` | Float | 历史90分位间隔 |
| `max_gap` | Int | 历史最大间隔 |
| `current_gap_percentile` | Float | 当前遗漏在历史中的百分位 |
| `recent_records` | List | 最近N次出现记录 |

### 6.2 统计计算公式

#### 6.2.1 出现频率

```text
occurrence_rate = occurrence_count / 历史总期数
```

#### 6.2.2 当前遗漏期数

```text
current_gap = 当前最新期号与该形态最近一次出现期号之间的期数差
```

#### 6.2.3 间隔序列

```text
gap_list = [第2次出现与第1次出现的间隔, 第3次与第2次出现的间隔, ...]
```

#### 6.2.4 当前遗漏百分位

```text
current_gap_percentile = 历史 gap_list 中 <= current_gap 的样本占比
```

### 6.3 展示型标签口径

对于常规高频形态，可展示“相对历史位置”标签：

| 条件 | 标签 |
|---|---|
| percentile < 50% | 偏低 |
| 50% ≤ percentile < 80% | 正常 |
| 80% ≤ percentile < 90% | 偏高 |
| percentile ≥ 90% | 显著偏高 |

**注意：该标签仅表示历史相对位置，不表示未来概率。**

## 7. 产品功能设计

## 7.1 首页

### 7.1.1 模块组成

1. 今日 / 最近开奖入口
2. 快速输入号码入口
3. 热门形态入口
4. 稀有形态入口

### 7.1.2 首页目标

---

## 7.2 历史开奖查询

### 7.2.1 功能说明

用户可按以下方式查询历史开奖：

- 最近N期
- 指定期号
- 指定日期
- 条件筛选（后续可选）

### 7.2.2 查询结果展示

每期开奖展示：

- 期号
- 开奖日期
- 红球号码
- 蓝球号码
- 当期主要形态标签（如大小比、奇偶比、连号、和值区间）

### 7.2.3 详情页扩展

单期开奖详情页展示：

- 该期所有基础形态
- 与上一期的重号 / 邻号
- 触发的特殊形态
- 该期可点击跳转到各形态详情页

---

## 7.3 用户输入号码形态识别

### 7.3.1 功能目标

当用户输入一组号码后，系统不做“预测”，而是识别该号码在历史统计体系中对应的形态，并展示其历史档案。

### 7.3.2 输入规则

- 红球：6个，1-33，不重复
- 蓝球：1个，1-16
- 支持只分析红球

### 7.3.3 输出内容

1. 该组号码命中的所有形态
2. 每个形态的历史出现次数
3. 每个形态的历史出现频率
4. 最近一次出现期号 / 日期
5. 当前遗漏期数
6. 历史平均 / 中位 / 分位间隔（适用于高频形态）
7. 是否属于低频 / 稀有形态

### 7.3.4 示例文案（允许）

- 你的号码触发了“全偶”形态。
- 该形态历史出现 18 次，占全部历史开奖的 1.2%。
- 最近一次出现于第 XXXX 期，距今已 XX 期。
- 该形态属于低频形态。

### 7.3.5 示例文案（禁止）

- 这种号码更难开出
- 这种号码更可能中奖
- 这种形态该回补了
- 建议避开 / 建议重点关注
- 低于一般可能性 / 高于一般可能性

---

## 7.4 常规形态档案

### 7.4.1 适用对象

适用于样本量充足、适合做历史分布展示的形态：

- 大小比
- 奇偶比
- 三区间分布
- 012路分布
- 常规连号结构
- 蓝球大小 / 奇偶 / 012路

### 7.4.2 页面展示字段

- 形态名称
- 历史出现次数
- 历史出现频率
- 最近一次出现
- 当前遗漏期数
- 历史平均间隔
- 历史中位间隔
- P80 / P90 间隔
- 当前遗漏相对历史位置标签
- 最近10次出现记录

### 7.4.3 文案规则

允许：

- “当前已间隔5期，处于历史较高区间（P88）”
- “历史平均间隔2.2期，中位间隔2期”

禁止：

- “很久没出了，建议关注”
- “大概率快出了”
- “回补机会增大”

## 7.5 特殊形态档案

### 7.5.1 适用对象

适用于低频、极端、稀有形态：

- 全6连号
- 全同尾
- 全奇 / 全偶
- 极端断区
- 极端和值
- 稀有 AC 值

### 7.5.2 页面展示字段

- 历史出现次数
- 首次出现期号 / 日期
- 最近一次出现期号 / 日期
- 当前遗漏期数

## 7.6 形态博物馆

### 7.6.1 模块定位

“形态博物馆”是一个聚合型浏览页面，用于提升用户停留时长与探索欲，但严格维持在“历史档案展示”范围内。

### 7.6.2 展示内容

- 稀有形态榜
- 久未出现形态榜
- 极端形态榜
- 最近一次出现最久远的形态
- 历史次数最少的形态

### 7.6.3 展示要求

- 榜单名称必须中性，如“历史低频形态”“历史罕见形态”

## 7.7 最近出现记录 / 时间轴

### 7.7.1 功能说明

每个形态详情页支持展示最近N次出现记录，建议 N=10。

### 7.7.2 记录字段

- 期号
- 日期
- 当期号码
- 对应形态值

### 7.7.3 产品价值

- 提高可信度
- 让用户能回看真实历史记录
- 降低“黑箱评分”感

## 9. 交互流程

## 9.1 用户输入号码分析流程

1. 用户输入红球和蓝球
2. 系统校验格式合法性
3. 系统解析该组号码命中的各类形态
4. 系统查询每个形态的历史档案
5. 返回形态识别结果与统计信息
6. 用户可点击单个形态进入详情页继续浏览

## 9.2 用户浏览历史形态流程

1. 用户从首页进入某个形态卡片
2. 进入形态详情页
3. 查看频率 / 最近出现 / 当前遗漏 / 历史分布
4. 查看最近10次出现记录
5. 继续跳转到相关相似形态

## 10. 数据与技术实现建议

### 10.1 统计计算策略

建议采用“离线预计算 + 在线查询”的组合：

- 每次开奖数据更新后，增量更新相关形态统计
- 前台查询不做全量扫描
- `shape_stats`、`draw_shape_map` 作为核心查询表

### 10.2 建议的数据表

#### 10.2.1 `draw_results`
开奖基础表

#### 10.2.2 `shape_definitions`
形态定义表，便于统一维护 shape_key 和展示文案

#### 10.2.3 `draw_shape_map`
开奖与形态的映射表

#### 10.2.4 `shape_stats`
形态聚合统计表

#### 10.2.5 `shape_recent_records`
各形态最近N次出现记录表

### 10.3 性能要求

- 首页卡片必须走缓存
- 形态详情页优先查询聚合表
- 仅最近记录回查明细表
- 输入号码分析接口目标响应时间 < 800ms（缓存命中情况下）

## 15. 附录：术语表

| 术语 | 说明 |
|---|---|
| 当前遗漏期数 | 当前最新一期距离该号码/形态最近一次出现的期数差 |
| 间隔 | 某形态两次连续出现之间的期数差 |
| 频率 | 该形态出现次数 / 历史总期数 |
| 百分位 | 当前遗漏在历史间隔分布中的相对位置 |
| 常规形态 | 样本量充足、适合做历史分布展示的形态 |
| 特殊形态 | 出现次数少、适合做档案展示的稀有形态 |
