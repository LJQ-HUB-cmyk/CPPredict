# 双色球个人分析台 评分公式、计算规则与名词解释


## 1. 说明

这份文档只描述当前网页里**实际落地**的计算逻辑，不讨论彩票学意义上的“真实概率”。

系统里有两类评分场景：

1. **当前号码评分 / 本期推荐评分**
   - 使用全历史样本。
   - 会受到你在“本期推荐”里勾选的排除项影响。
2. **历史开奖详情里的“截止当时评分”**
   - 只使用“从第一期到该期开奖当期”为止的历史数据。
   - **不套用**当前推荐区的勾选排除项。
   - 目的是回答：**这组号码在当时那个历史截面下，结构上属于什么水平。**

## 2. 基础号码定义

### 2.1 红球大小比
- 小号：`1~16`
- 大号：`17~33`
- 大小比 = 6 个红球里小号个数 : 大号个数

例：`01 07 12 18 25 32` → 小 3 个、大 3 个 → `3:3`

### 2.2 红球奇偶比
- 奇偶比 = 6 个红球里奇数个数 : 偶数个数

### 2.3 三区分布
- 1 区：`1~11`
- 2 区：`12~22`
- 3 区：`23~33`
- 三区比 = 三个区间各有几个红球，格式为 `z1-z2-z3`

例：`01 07 12 18 25 32` → `2-2-2`

### 2.4 断区
- 若某一区红球数为 `0`，则视为断区。
- 可出现：`断1区`、`断2区`、`断3区`
- 若三个区都有号，则记为 `不断区`
- 多断区时用 `+` 连接

### 2.5 012 路分布
- 按 `n mod 3` 分为 `0 路 / 1 路 / 2 路`
- 统计 6 个红球里三类的数量，记为 `m0-m1-m2`

### 2.6 尾数组型
按红球个位数分组：
- 若 6 个尾数全不同 → `全不同尾`
- 若 6 个尾数完全相同 → `全同尾`
- 否则，统计所有“重复尾数组”的组大小，例如：
  - 1 个 2 同尾 → `1组2同尾`
  - 2 个 2 同尾 → `1组2同尾+1组2同尾`
  - 1 个 3 同尾 → `1组3同尾`

### 2.7 连号结构
先按升序扫描红球，识别所有长度 ≥2 的连续段：
- 没有连续段 → `无连号`
- 一段 2 连 → `1组2连`
- 一段 3 连 → `1组3连`
- 两段 2 连 → `2连+2连`
- 一段 3 连加一段 2 连 → `3连+2连`
- 六个连续 → `全6连号`

### 2.8 和值
- `sum = r1 + r2 + r3 + r4 + r5 + r6`

### 2.9 和值区间
- `sum_bucket = floor(sum / 10) * 10`
- 展示为 `[lo, lo+9]`
- 下限最小钳制到 `20`
- 上限最大钳制到 `189`

例：和值 `87` → `80-89`

### 2.10 跨度
- `span = max(reds) - min(reds)`

### 2.11 跨度区间
- `span_bucket_lo = floor(span / 5) * 5`
- 展示为 `[lo, lo+4]`

例：跨度 `22` → `20-24`

### 2.12 AC 值
先求 6 个红球所有两两正差值的去重集合：

- 差值集合大小：`|D|`
- 红球数固定为 6，因此：
- `AC = |D| - (6 - 1) = |D| - 5`

更形式化地写：

`D = { ri - rj | i > j }`

`AC = |D| - 5`

AC 越低，说明整体越规则、越“挤”；AC 越高，说明整体越离散。

## 3. 红蓝联动类指标

### 3.1 蓝球大小
- 小：`1~8`
- 大：`9~16`

### 3.2 蓝球奇偶
- 蓝球奇或偶

### 3.3 蓝球 012 路
- `blue mod 3`

### 3.4 红蓝同号
- 若蓝球数值与任一红球相同，则为 `红蓝同号`
- 否则为 `红蓝不同号`

### 3.5 红蓝最小差
- `rbMin = min(|red_i - blue|)`
- 分桶：
  - `0`
  - `1`
  - `2`
  - `3+`

### 3.6 红蓝奇偶联动
- 形式：`蓝{奇/偶} + 红奇偶比 {odd}:{even}`

### 3.7 红蓝大小联动
- 形式：`蓝{小/大} + 红大小比 {small}:{big}`

### 3.8 红蓝 012 联动
- 形式：`蓝{k}路 + 红012比 {m0}-{m1}-{m2}`

## 4. 对上一期关系

### 4.1 重号（repeat_count）
- 与上一期开奖相比，红球完全相同的个数

### 4.2 邻号（neighbor_count）
- 与上一期开奖任一红球相邻（`±1`）的当前红球个数
- 不计入已经是重号的那些号码

### 4.3 对最新一期重号 / 邻号
在“当前号码评分 / 本期推荐”里，系统还会额外算：
- 与**最新一期**红球重合了几个
- 与**最新一期**红球相邻了几个

这个指标主要用于当前选号参考，不用于历史开奖“截止当时评分”的惩罚项。

## 5. 特殊形态定义

只要命中以下任一条件，就会追加 `special_shape` 条目：

- `全奇`：6 红全奇
- `全偶`：6 红全偶
- `全同尾`：尾数全相同
- `全6连号`
- `断区形态`：任一区为空
- `极低和值`：`sum <= 60`
- `极高和值`：`sum >= 140`
- `极低AC值`：`AC <= 2`
- `极高AC值`：`AC >= 9`
- `罕见连号结构`：属于以下之一
  - `1组5连`
  - `1组4连`
  - `3连+2连`
  - `4连+2连`

注意：这些特殊形态既会用于历史档案展示，也会用于推荐过滤和评分惩罚。

## 6. 历史档案统计公式

设历史总期数为 `N`，某形态在历史里命中的期位序列为：

`positions = [p1, p2, ..., pk]`

其中 `k = occurrence_count`

### 6.1 出现次数
- `occurrence_count = k`

### 6.2 出现频率
- `occurrence_rate = occurrence_count / N`

### 6.3 当前遗漏
- `current_gap = (N - 1) - positions[-1]`

即：距离最近一次出现已经过去多少期。

### 6.4 历史间隔序列
- `gaps[j] = positions[j] - positions[j-1]`
- 只在 `j >= 1` 时成立

### 6.5 平均遗漏
- `avg_gap = mean(gaps)`

### 6.6 中位遗漏
- `median_gap = median(gaps)`

### 6.7 P80 / P90
- `p80_gap = percentile(gaps, 0.8)`
- `p90_gap = percentile(gaps, 0.9)`

### 6.8 最大遗漏
- `max_gap = max(gaps)`

### 6.9 当前遗漏相对位置（current_gap_percentile）
如果 `gaps` 为空，标记为“样本不足”。
否则：

`current_gap_percentile = count(g <= current_gap) / len(gaps) * 100`

即：当前遗漏在它自己历史间隔分布里，大概处在什么位置。

### 6.10 当前遗漏位置标签（gap_position_label）
- 样本不足：`gaps` 为空
- `偏低`：`percentile < 50`
- `正常`：`50 <= percentile < 80`
- `偏高`：`80 <= percentile < 90`
- `显著偏高`：`percentile >= 90`

### 6.11 稀有度标签（rarity_label）
按历史出现次数划分：
- `极罕见`：`occurrence_count <= 3`
- `罕见`：`occurrence_count <= 10`
- `低频`：`occurrence_count <= 30`
- `常规`：`occurrence_count > 30`

## 7. “历史常见度”评分公式（prevalence）

这是系统三大分数之一。

### 7.1 先取当前号码命中的全部形态
系统会把一组号码展开成多条形态引用，例如：
- 大小比
- 奇偶比
- 三区比
- 断区
- 012 路
- 尾数组型
- 连号结构
- 和值区间
- 跨度区间
- AC 值
- 蓝球大小/奇偶/012
- 红蓝联动
- 重号 / 邻号

### 7.2 排除特殊形态后参与常见度计算
常见度计算时，**不把 `special_shape` 纳入平均值**。

### 7.3 单个形态的“常见度分位”
对每一个 `type`（例如 `size_ratio`、`odd_even_ratio`）单独做排序：

- 按该 type 下各 shape 的历史出现次数升序排序
- 若次数相同，用 `display` 字典序打破并列
- 排名归一化到 `[0,1]`

公式可以写成：

`shape_percentile = rank / (count_of_shapes_in_type - 1)`

其中 rank 从 0 开始。

含义：
- 越接近 `1`，说明这个形态在本类型里越常见
- 越接近 `0`，说明这个形态在本类型里越少见

### 7.4 总常见度
对所有非特殊形态的分位值取平均：

`prevalence = round(avg(shape_percentile_list) * 100)`

即得到 `0~100` 的“历史常见度”。

## 8. “结构均衡度”评分公式（balance）

也是系统三大分数之一。

## 8.1 大小分
使用固定查表：

| 小号个数 | 分数 |
|---|---:|
| 0 | 20 |
| 1 | 52 |
| 2 | 80 |
| 3 | 100 |
| 4 | 80 |
| 5 | 52 |
| 6 | 20 |

### 8.2 奇偶分
同样使用固定查表：

| 奇数个数 | 分数 |
|---|---:|
| 0 | 20 |
| 1 | 52 |
| 2 | 80 |
| 3 | 100 |
| 4 | 80 |
| 5 | 52 |
| 6 | 20 |

### 8.3 三区分布分
令：

`zoneDiff = |z1 - 2| + |z2 - 2| + |z3 - 2|`

则：

`zoneScore = clamp(100 - zoneDiff * 22, 24, 100)`

三区越接近 `2-2-2`，得分越高。

### 8.4 和值 / 跨度 / AC 的距离分
这三项都用同一个函数：`valueScoreByDistance(value, p10, p50, p90)`

先基于历史样本算：
- `p10`
- `p50`
- `p90`

然后：

若 `value <= p50`：

`score = clamp(round(100 - |value - p50| / max(1, p50 - p10) * 38), 20, 100)`

若 `value > p50`：

`score = clamp(round(100 - |value - p50| / max(1, p90 - p50) * 38), 20, 100)`

含义：
- 越靠近历史中位数 `p50`，分数越高
- 离得越远，分数越低
- 最低钳制到 `20`

得到：
- `sumScore`
- `spanScore`
- `acScore`

### 8.5 尾数组型分
- `全不同尾` → `96`
- `全同尾` → `8`
- 其他情况 → `72`

### 8.6 连号结构分
- 默认 `90`
- `1组2连` → `80`
- `2连+2连` → `72`
- `1组3连` → `58`
- 以下极端长连结构 → `16`
  - `1组4连`
  - `1组5连`
  - `全6连号`
  - `3连+2连`
  - `4连+2连`

### 8.7 号码热度分（红球）
令 `redFreq[n]` 是号码 n 在历史中出现次数，`maxRedFreq` 是 33 个红球里出现次数最大者。

`numberFreqScore = clamp(round(avg(redFreq[n] / maxRedFreq) * 100), 20, 100)`

这里对 6 个红球先算相对热度，再求平均。

### 8.8 蓝球热度分
令 `blueFreq[blue]` 为当前蓝球历史次数，`maxBlueFreq` 为 16 个蓝球里历史最高次数。

`blueFreqScore = clamp(round(blueFreq[blue] / maxBlueFreq * 100), 20, 100)`

### 8.9 最终均衡度
按固定权重加权：

`balance = round(`

`sizeScore * 0.16 +`
`oddEvenScore * 0.16 +`
`zoneScore * 0.20 +`
`sumScore * 0.14 +`
`spanScore * 0.09 +`
`acScore * 0.09 +`
`tailScore * 0.06 +`
`runScore * 0.05 +`
`numberFreqScore * 0.03 +`
`blueFreqScore * 0.02`

`)`

## 9. 当前号码 / 推荐号码的惩罚项与综合评分

### 9.1 规则命中（ruleHits）
如果你在推荐区勾选了过滤规则，命中对应条件就会产生 penalty：

| 规则 | 触发条件 | penalty |
|---|---|---:|
| 排除全奇/全偶 | odd=6 或 even=6 | 16 |
| 排除全同尾 | tailPattern=全同尾 | 18 |
| 排除全6连号 | runLabel=全6连号 | 20 |
| 排除罕见长连号 | 1组5连/1组4连/3连+2连/4连+2连 | 16 |
| 排除断区 | breakParts 非空 | 14 |
| 排除极低和值 | sum <= 60 | 12 |
| 排除极高和值 | sum >= 140 | 12 |
| 排除极低 AC | ac <= 2 | 12 |
| 排除极高 AC | ac >= 9 | 10 |
| 排除红蓝同号 | blue 出现在 reds 中 | 9 |
| 排除历史完全重复 | 历史里已出现完全相同红蓝组合 | 22 |
| 避免与最新一期高重合 | 与最新一期红球重合数 >= 4 | 10 |
| 优先均衡结构 | balance < 62 | 8 |

### 9.2 扣分后保留分
`penaltyScore = clamp(100 - totalPenalty, 0, 100)`

### 9.3 当前号码 / 推荐号码综合评分
`score = clamp(round(prevalence * 0.52 + balance * 0.40 + penaltyScore * 0.08 - 2 * ruleHitsCount), 1, 99)`

说明：
- 历史常见度权重最高：`52%`
- 结构均衡度次之：`40%`
- 惩罚后保留分：`8%`
- 另外每命中一条 ruleHit，再额外减 `2` 分

### 9.4 等级
- `S`：`score >= 85`
- `A`：`76 ~ 84`
- `B`：`67 ~ 75`
- `C`：`58 ~ 66`
- `D`：`<= 57`

## 10. 历史开奖详情里的“截止当时评分”

历史开奖详情里新增的三个分数：
- 综合评分
- 历史常见度
- 结构均衡度

它们不是用“全历史回看”算的，而是：

**只用从第一期到该期开奖当期为止的数据，做一次当时视角的重算。**

### 10.1 当时常见度
- 公式与第 7 节相同
- 但历史样本窗口改为：`[第一期, 当前这期]`

### 10.2 当时均衡度
- 公式与第 8 节相同
- 其中 `p10 / p50 / p90`、红球频次、蓝球频次，也都只使用当时窗口

### 10.3 当时综合评分
历史开奖详情**不使用当前推荐区勾选项**，因此不做惩罚过滤：

`historical_score = clamp(round(prevalence * 0.52 + balance * 0.40 + 8), 1, 99)`

这里的 `+8`，等价于把 `penaltyScore` 固定看成 `100`，即：

`100 * 0.08 = 8`

所以它本质上是一个**基线综合分**。

## 11. 推荐逻辑

推荐不是“追号”“冷热回补”模型，而是**随机采样 + 结构过滤 + 综合打分**。

### 11.1 采样次数
- 勾选“优先均衡结构”时：随机采样 `9000` 次
- 否则：随机采样 `6500` 次

### 11.2 候选生成
每次候选：
- 红球：从 `1~33` 随机打乱后取 6 个，不重复
- 蓝球：从 `1~16` 随机 1 个

### 11.3 候选过滤
对每个候选先算 `candidateProfile`：
- 只要 `ruleHits.length > 0`，直接淘汰
- 若红蓝组合 key 重复，也淘汰

### 11.4 候选排序
先按：
1. `score` 降序
2. `balance` 降序
3. `key` 字典序

### 11.5 差异化选 5 组
第一轮优先选：
- 与已选任一组的红球重合数都 `<= 3`

如果不足 5 组，再从剩余高分候选里补足到 5 组。

### 11.6 展示排序
推荐结果支持以下排序：
- 按综合评分
- 按历史常见度
- 按结构均衡度
- 按相对冷门优先

其中“相对冷门优先”即 `prevalence` 升序。

## 12. 正向标签（reasons）生成规则

系统会给当前号码/推荐号码补几条“正向标签”，用于 UI 解释，不直接参与打分。

满足以下条件时会追加 reasons：
- `prevalence >= 75` → `历史形态偏常见`
- `balance >= 78` → `结构均衡度较高`
- 与最新一期重号数 `<= 2` → `与最新一期重合较低`
- 未触发任何特殊形态 → `未触发特殊形态`
- 三区正好 `2-2-2` → `三区分布均衡`
- 奇偶 `3:3` 且大小 `3:3` → `奇偶/大小接近中性`
- 如果上述都没触发 → `通过当前评分逻辑`

## 13. 形态博物馆分组规则

### 13.1 历史低频形态
按以下键排序后取前 18 条：
1. `occurrence_count` 升序
2. `current_gap` 降序
3. `display` 升序

### 13.2 当前遗漏偏高形态
先筛掉：
- `occurrence_count < 5`
- `current_gap_percentile is None`

剩余形态按以下键排序后取前 18 条：
1. `current_gap_percentile` 降序
2. `current_gap` 降序
3. `display` 升序

### 13.3 最近一次出现最久远的形态
按以下键排序后取前 18 条：
1. `last_seen_date` 升序
2. `occurrence_count` 升序

### 13.4 特殊档案精选
先取 `kind == special` 的条目，再按以下键排序后取前 18 条：
1. `occurrence_count` 升序
2. `current_gap` 降序

## 14. 你最该注意的边界

### 14.1 这些分数不是命中概率
它们表达的是：
- 这组号码在历史结构上常不常见
- 结构是否更接近系统偏好的“中性/均衡”形态
- 在你当前过滤策略下，会不会被判为不想要的组合

### 14.2 “常见度高”不等于“更该买”
常见度只是“历史形态排序位置”高，不是未来发生概率承诺。

### 14.3 “截止当时评分”比“全历史回看评分”更合理
因为它避免了把未来数据泄漏给过去的开奖。
这也是你这次新增功能真正有价值的地方。

## 15. 当前版本最核心的三个公式

### 15.1 常见度
`prevalence = round(avg(type_percentile_of_non_special_shapes) * 100)`

### 15.2 均衡度
`balance = round(weighted_sum_of_size_odd_zone_sum_span_ac_tail_run_heat_scores)`

### 15.3 综合分
当前号码 / 推荐号码：

`score = clamp(round(prevalence * 0.52 + balance * 0.40 + penaltyScore * 0.08 - 2 * ruleHitsCount), 1, 99)`

历史开奖“截止当时评分”：

`historical_score = clamp(round(prevalence * 0.52 + balance * 0.40 + 8), 1, 99)`
